TL;DR & Key Findings

Platform Snapshot (Dec 2025)

The platform shifted in 2024–2025. React Native 0.76 (Oct 2024) made the New Architecture the default. RN 0.83 (Dec 2025) ships on React 19.2 with zero breaking changes. React Compiler 1.0 went stable Oct 7, 2025 with full React Native support.

Staff+ engineers must now reason about Fabric / TurboModules / JSI as defaults, not opt-ins. Paper is legacy code-debt. The job is no longer "ship features in JS" — it is owning a C++/JSI/native interop platform that 600M+ users depend on.

The platform consolidated around Meta + Microsoft + Shopify + Expo + Callstack + Software Mansion, governed by the new React Foundation announced at React Conf 2025.

Finding 01

Architecture is settled, migration is the work. The Bridge is being deleted. RN 0.82 was "the first release built entirely on the New Architecture, with no fallback." Paper-only modules in 2026 = rewrite within one major version.

Finding 02

Performance is now a C++ story. JSI, Hermes Hades (up to 30× shorter GC pauses), Fabric's C++ shadow tree, and Reanimated worklets all live below the JS layer. Staff engineers must read C++/Obj-C++/JNI.

Finding 03

Storage has diversified. MMKV (≥30× faster than AsyncStorage) is the new default for KV. WatermelonDB for sync. PowerSync + TanStack DB is the 2025 local-first stack. AsyncStorage is legacy.

Finding 04

OTA is no longer free. App Center/CodePush retired March 31, 2025. Options: EAS Update (dominant), self-hosted CodePush server, or Stallion/Revopush. Staff must own runtime-version policy and rollback.

Finding 05

Testing has bifurcated. Maestro wins on developer velocity. Detox (<2% flake) remains for deep RN sync. Test pyramid: Jest+RNTL 70%, MSW integration 20%, E2E 10%.

Finding 06

Security has a new Top 10. OWASP Mobile 2024 adds M2 Inadequate Supply Chain Security. SSL pinning, Play Integrity/App Attest, MMKV/Realm encryption are table stakes.

Finding 07

AI tooling is a production concern. MCP is now an industry standard. Flip side: GitClear found 8× rise in duplicated code blocks; CodeRabbit found AI code has 1.7× more bugs.

⚠ Action Thresholds

Freeze features when: crash-free sessions <99.5% · cold start P95 >4s · major dep unmaintained ≥6mo · AI-generated PR rate >30%

01 — Android Native Engineering

Android Native Engineering

Activity/Fragment Lifecycle

Activity states: Created → Started → Resumed → Paused → Stopped → Destroyed, plus the process-death path where onSaveInstanceState(Bundle) is the only persistence guarantee. Config changes (rotation, locale, dark mode) recreate the Activity unless android:configChanges opts out — RN's MainActivity declares configChanges="keyboard|keyboardHidden|orientation|screenSize|uiMode".

Fragment lifecycle adds onAttach / onCreateView / onViewCreated / onDestroyView / onDetach. The classic IllegalStateException: Can not perform this action after onSaveInstanceState — fix with commitAllowingStateLoss() only when the UI is truly disposable.

Services & Background Restrictions

Foreground services on Android 14 (API 34) require a foregroundServiceType and matching runtime permission. Background execution is dead for free-form work — use WorkManager with constraints or JobScheduler. Long-running JS like background uploads must run inside a native foreground service with a JSI/TurboModule shim, not a JS setInterval.

App Startup

Cold start: Zygote forks → Application.attachBaseContext → ContentProvider.onCreate (Firebase/Crashlytics auto-init providers, often hundreds of ms) → Application.onCreate → Activity.onCreate. AndroidX App Startup library collapses N ContentProviders into one. RN 0.76 reduced libraries to a single merged libreactnative.so: app size reduced ~3.8MB (20%) and median startup ~15ms (~8%).

ANRs

Triggered by main-thread block ≥5s for input events or ≥10s for broadcast receivers. system_server detects via SIGQUIT and writes trace to /data/anr/traces.txt. For RN the typical culprit is JS-thread starvation, not main thread blocking itself. Use StrictMode to surface disk/network on main thread.

⚠ Production Pitfall — R8 & TurboModules

Enabling minifyEnabled true in release without verifying codegen-generated TurboModule classes are kept causes ClassNotFoundException only in release. Keep rules live in node_modules/react-native/proguard-rules.pro — augment for every native module using @ReactMethod reflection.

Kotlin Coroutines

CoroutineScope + SupervisorJob is the structured-concurrency primitive. Dispatchers: Main for UI, IO for blocking I/O (64-thread pool), Default for CPU-bound. For RN TurboModules: launch on Dispatchers.IO, switch to Dispatchers.Main only when callbacks into JS need it.

02 — iOS Native Engineering

iOS Native Engineering

App & ViewController Lifecycle

UIApplication states: Not Running → Inactive → Active → Background → Suspended. RN 0.78 (Feb 2025) switched to a Swift-based AppDelegate, moving away from Objective-C after 11 years. UIViewController: loadView → viewDidLoad → viewWillAppear → viewIsAppearing (iOS 13+) → viewDidAppear. Fabric mounts via UIViews wrapped in RCTViewComponentView.

ARC & Retain Cycles

Closures capturing self strongly inside Combine subscriptions or RxSwift chains are the classic leak. Always use [weak self] in escaping closures that outlive the call site. unowned crashes on access after deallocation — only use when lifetime is provably tied to self.

🔴 Deadlock Risk

A dispatch_sync from a TurboModule on the JS thread back to the main thread, where the main thread is also waiting on the JS thread, deadlocks the app. Always use dispatch_async from TurboModules.

Keychain & Biometric Auth

For refresh tokens use kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly — survives reboot, doesn't sync, doesn't backup. Pair biometric auth (LAContext) with a Keychain item bound by SecAccessControlCreateWithFlags(.biometryCurrentSet) so adding a new fingerprint invalidates the key — defends against enrollment attacks.

RN ↔ iOS Integration (New Arch)

TurboModules in Swift: Swift class with @objc methods → Obj-C++ (.mm) shim conforming to Codegen-generated *Spec protocol → C++ *SpecJSI registered with TurboModule registry. Margelo's Nitro Modules (2025) collapse the Obj-C++ shim via Swift's new C++ interop directly.

APNs Internals

Device token is 64-byte hex tied to device + bundle ID + APNs environment. Background push uses content-available: 1 and apns-priority: 5 — iOS coalesces these under low-power conditions and may drop them. Notification Service Extensions have a 30-second budget and ~24MB memory limit.

03 — React Internals

React Internals (18 & 19)

Fiber Architecture

A FiberNode has pointers: return (parent), child, sibling, alternate. React maintains two trees: current (committed) and work-in-progress (WIP). The reconciler builds WIP via beginWork (compute children) and completeWork (collect side-effects); the commit phase applies them.

Render is pure and interruptible — Strict Mode calls it twice in dev to catch impurity. Commit is synchronous and runs side effects (useLayoutEffect, Fabric mutations, refs).

React Compiler 1.0 (Oct 7, 2025)

Meta reports up to 12% faster initial loads and 2.5× faster interactions in the Meta Quest Store. Expo SDK 54 enables it out of the box. The compiler memoizes automatically — manual useMemo/useCallback become escape hatches for code that violates the Rules of React.

ℹ Compatibility Note

Reanimated's useSharedValue is special-cased. See the agreed-upon contract at github.com/reactwg/react-compiler/discussions/14 before adopting the compiler in Reanimated-heavy codebases.

Automatic Batching (React 18+)

Batches all state updates in a single event tick — including Promise callbacks, setTimeout, native event handlers. N setStates in a TurboModule callback cause one re-render, not N.

04 — JavaScript Engine Internals

JavaScript Engine Internals

Hermes & Hades GC

Bytecode (.hbc) is precompiled at build time — no JS parse on device. The current default GC is Hades, a concurrent OG mark-sweep: "up to 30× shorter GC pause time on 64-bit devices and 14× on 32-bit." Hades runs OG concurrently with the interpreter, accepting slightly slower throughput for radically lower pause times. Hermes V1 (preview in RN 0.83) may introduce JIT compilation — opt-in via flag.

JSI vs Old Bridge

Old BridgeJSI (New Architecture)
SerializationJSON stringify + deserializeZero serialization
ThreadingAsync onlySynchronous when needed
InvocationJava/ObjC reflectionDirect C++ std::function call
MMKV benefit~242ms for 1000 reads~12ms — 30× speedup

Event Loop

Call stack → task → microtask drain → render. Microtasks (Promise.then, queueMicrotask) run after every task; macrotasks (setTimeout, MessageChannel) yield to the loop. In RN, requestAnimationFrame in JS is a setTimeout(0) polyfill unless inside a Reanimated worklet, where the worklet runtime hooks the Choreographer/CADisplayLink directly.

05 — Distributed Systems

Distributed Systems for Mobile

Auth

OAuth2 Authorization Code + PKCE is the only acceptable mobile flow (no client secret). Refresh-token rotation with reuse detection (RFC 6749 §10.4) — invalidate the entire family on reuse. Never store JWTs in AsyncStorage — use Keychain (iOS) / EncryptedSharedPreferences or Keystore-wrapped storage (Android), or expo-secure-store.

CDN & Load Balancing

L4 (TCP, lower latency) vs L7 (HTTP/2, route by path/header). Mobile benefits from connection coalescing (HTTP/2) avoiding cold-start TLS handshakes per request. EAS Update ships JS bundles via Cloudflare's global edge. Always version bundles by content hash — never mutate a URL.

WatermelonDB Sync Protocol

Per-column client-wins strategy. Server must support: pull endpoint (returns changes since lastPulledAt + server time) and push endpoint (accepts changes, returns 409 on conflict).

🔴 Critical — Soft Deletes

Soft deletes are mandatory. WatermelonDB's sync protocol requires deleted records in the pull response. Hard deletes break sync entirely. Client-side IDs have ~8×10²⁴ possible combinations — no coordination needed.

PowerSync + TanStack DB (2025 Stack)

PowerSync syncs Postgres → SQLite on device with reactive queries. TanStack DB 0.6 adds persistence, offline support, hierarchical data with sub-millisecond live queries via differential dataflow. The combo gives optimistic updates with automatic rollback and cross-collection joins, running on RN via @powersync/react-native + @powersync/op-sqlite.

08 — Software Architecture

Software Architecture Patterns

Clean Architecture in RN

Three layers: domain/ (entities, use-cases, pure TS — no React, no Native), data/ (repositories, API clients, DB), presentation/ (components, hooks, screens). Domain depends on nothing; data depends on domain; presentation depends on both via interfaces.

  ui/              # Shared components (RN + RN-web)
  domain/          # Pure TS business logic
  api-client/      # Generated from OpenAPI/GraphQL
  config/          # ESLint, TS, Prettier configs

Super-App Architecture

Module federation (Re.Pack/MF) lets a host app load remote JS bundles at runtime. Risk: native modules must be pre-bundled in the host — RN can't dynamically link native code. Module federation only swaps JS.

09 — Brownfield Architecture

Brownfield React Native Architecture

Integration Pattern

Embed via RCTRootViewFactory (iOS) and ReactFragment/ReactDelegate (Android). Native shell owns navigation; RN screens are leaf views. Microsoft Office uses this pattern: 40+ experiences use RN for cross-platform features serving 600M users.

Communication

TurboModules for native→JS function calls; NativeEventEmitter for JS subscribing to native events. Pass auth tokens via a TurboModule reading from Keychain/Keystore — never via props or query strings.

OTA in Brownfield

Only the JS bundle and JS-side assets are OTA-updatable. Native modules, native UI, and the bridge itself require a binary release. Use EAS Update with a strict runtime-version policy that bumps on every native change — Expo's fingerprint tool auto-detects this.

Performance

Cold start of RN surface: ~200–500ms on mid-tier Android for the first JS load. Solution: pre-warm at app startup (one-time JS init), lazy-load route bundles via inline-requires.

10 — Production Incidents

Production Incidents & Debugging

Crash Debugging

Sentry: upload sourcemaps with the build, symbolicate JS stack to source. Native crashes need .dSYM (iOS) or unstripped .so (Android); Hermes adds .hbc symbol files. ANR debugging: pull /data/anr/traces.txt from device or adb shell dumpsys dropbox. Look for the main thread (tid=1) — if blocked on JNI in MessageQueueThreadImpl, the JS thread is starving.

Discord / Reanimated Jank on Android New Architecture Post-Mortem

Symptom: Animations felt like "a 2006 PowerPoint slideshow" after migrating to Fabric.

Root cause: Reanimated was triggering layout-prop commits on every frame via shadowTree.commit(), causing 30+ ms commit times.

Fix (Margelo): Two opt-in flags — FORCE_REACT_RENDER_FOR_SETTLED_ANIMATIONS and ANDROID_SYNCHRONOUSLY_UPDATE_UI_PROPS — bypass the commit pipeline for non-layout props.

Result: Jank frame rate on Discord Android New Arch reduced by 26%.

OTA Rollback

EAS: eas update --republish ships the previous good update on top. Always pair with a kill-switch feature flag (Statsig/LaunchDarkly) so you can disable a bad code path without waiting for OTA propagation.

Postmortem Template

Timeline (UTC) → Impact (users, $) → Root cause(s) → Contributing factors → Resolution → Action items with owners + due dates → Lessons. Track action items to closure — missing this is the #1 reason postmortems don't prevent recurrence.

15 — System Design

Advanced Mobile System Design

1. Realtime Chat (WhatsApp/Discord Scale)

Client (RN) ── WebSocket ──► LB (L7, sticky by user_id hash) │ Gateway Service (Go/Elixir) │ ┌──────────────────┼──────────────────┐ ▼ ▼ ▼ Message Bus Presence Auth/Session (Kafka) (Redis) (Redis) │ ▼ Storage (Scylla/Cassandra per-channel) │ ▼ Push Fanout (Kafka → APNs/FCM workers)

Ordering: server-assigned monotonic seq per channel. Offline queue: outbox table (WatermelonDB/MMKV) with retry+backoff, idempotency key (client_message_id UUID v7). Media: pre-signed S3 URLs from a dedicated upload service — client uploads directly, server only sees the URL.

2. Analytics Pipeline

Client SDK batches events (size or time threshold) → compressed POST → API Gateway → Kinesis/Kafka → enrichment Lambda → S3/BigQuery. Privacy: hash PII client-side; respect ATT/consent; never log free-text fields.

3. Push at Scale (100M Users)

Partition by device-token prefix; rate-limit per APNs connection; respect 429 Retry-After. Silent push uses content-available:1 (iOS) / data-only (FCM) — both throttled by OS. Use content-available sparingly or iOS will stop delivering them.

4. CRDT Collaboration (Notion/Figma Style)

Yjs as in-memory CRDT; persistence via expo-sqlite on RN; transport via y-websocket or Liveblocks/Hocuspocus. Use cases on RN: collaborative whiteboards, multi-user forms, shared document editing.

16 — Performance Engineering

Advanced Performance Engineering

FlashList v2 Internals

Rebuilt from the ground up for New Architecture. Adaptive draw buffer based on scroll velocity; eliminates estimatedItemSize requirement; up to 50% reduced blank area while scrolling vs v1. Moves to a JS-only solution with full New Arch support.

ℹ FlashList Performance Tips

Avoid key prop on item components — it breaks recycling. Use getItemType for heterogeneous lists. Use the useMappingHelper hook when mapping inside an item.

Reanimated 3/4 Worklet Architecture

Worklets are JS functions compiled by Babel into serializable objects executed on the UI runtime (a separate Hermes instance on the UI thread). Shared values bridge JS and UI runtimes — reads/writes from JS are async, from UI are sync. In RN 0.85+ behind a flag: a shared animation backend co-developed with Meta.

Startup Optimization Checklist

  • Inline requires (lazy module evaluation)
  • Hermes bytecode — default and always-on
  • Single libreactnative.so (RN 0.76)
  • Baseline Profiles on Android — up to 40% faster launch
  • Mergeable libraries on iOS (Xcode 15+)
  • Pre-warm RN runtime at app startup (for brownfield)

Target: P50 cold start <1.5s on Pixel 6a / iPhone 12; P95 <3s.

17 — Interview Bank

Interview Question Bank

Senior Level

  1. JSI vs Bridge for MMKV: Bridge serializes JSON across threads; JSI exposes the JS runtime to C++ via HostObjects — synchronous, direct calls. MMKV's ~30× speedup comes from synchronous JSI reads with zero serialization overhead.
  2. How does Fabric commit work? React renders WIP fiber tree → host config emits mount instructions → Fabric C++ shadow tree updates → diffed → applied to UI thread via UIManager.
  3. What is Hades? Concurrent mark-sweep old-gen collector running GC on a background thread. Up to 30× shorter GC pause times vs stop-the-world.
  4. Why is FlashList faster than FlatList? View recycling instead of create/destroy; precomputed item layouts; sync layout corrections in native; adaptive draw buffer.

Staff Level

  1. Design a CodePush migration for 10M MAU: Audit native modules for runtime-version impact → pick EAS Update vs self-host based on infra/cost → set channel→branch mapping → ship in parallel with kill-switch → cut over once stable.
  2. Crash-free sessions dropping from 95%: Sentry top issues by affected sessions → correlate to release diff → check OS/device distribution → reproduce on lowest-end device → bisect by release.
  3. For vs against React Compiler: For: auto-memoization, Meta-tested. Against: Reanimated requires special handling, one more build-time dep. Recommendation: enable in greenfield via ESLint rule; opt-in per-file in brownfield.

System Design

Design Instagram Stories on RN. Discuss: CDN strategy for video, prefetching the next story while showing the current, progressive WebP/AVIF, gesture handling via RNGH, offline view (store viewed-stories locally in MMKV), analytics batching, push for new-story-from-followed-user.

Recommendations

⚡ Tier 1 — Next 30 Days

1. Audit New Architecture Readiness

Upgrade to RN 0.76+, enable New Arch, resolve library compatibility. If >20% of direct deps are Paper-only, drive PRs to those libraries first.

2. Migrate Off App Center CodePush

Retired March 31, 2025. Choose: EAS Update (default), self-hosted CodePush (≥1 FTE), or Stallion. Ship a kill-switch feature flag.

3. Replace AsyncStorage with MMKV

Add an ESLint rule banning @react-native-async-storage/async-storage outside a legacy folder.

4. Replace FlatList with FlashList v2

Any list with >100 items or performance complaints. Remove key props from item components.

📅 Tier 2 — Next Quarter

5. OWASP MASVS Compliance

SSL pinning, encrypted storage, Play Integrity + App Attest, remove hardcoded secrets from JS bundle.

6. Wire MCP into Dev Tooling

Cursor or Claude Code with GitHub, Sentry, Postgres, Linear/Jira servers. Document team norms for AI-assisted PRs.

7. Performance Budgets in CI

Cold start P50/P95, JS thread idle %, app-size delta, frame-time P99. Use Flashlight or Maestro Cloud for automated capture.

8. Adopt Maestro for E2E

Cover top 10 user journeys; integrate into PR CI with parallel device runs.

🗓 Tier 3 — Next Year

9. Write a Technical Vision Document (Mobile Platform 2027)

Get exec alignment. Define quarterly checkpoints. Use it to say no to work that doesn't move toward the vision.

10. Hire a Mobile Platform Tech Lead

To own CI, native modules, design system, and observability. Without this role you pay a tax on every team's velocity forever.

11. Adopt React Compiler 1.0 Incrementally

Gate via ESLint preset; measure render reductions; expand from greenfield to brownfield over time.

12. Evaluate Hermes V1

Preview in RN 0.83. When stable, expect another step-change in startup time and CPU efficiency.

Caveats & Methodology Notes

Facts current as of December 2025 / early 2026. Verify version numbers against official changelogs before citing in production decisions. Single-vendor benchmarks (MMKV 30×, FlashList 5–10×) are correct in tested conditions but vary by device and scenario — reproduce in your environment.

The Microsoft Office "600M users" figure counts users of Office containing RN-built features (40+ surfaces), not users of fully-RN Office apps. React Native 1.0 was teased in 2025 but not shipped — plan as if 0.x continues through 2026.

AI-assisted engineering is a moving target. The principles — verification, scoped permissions, MCP as the integration layer — will outlast the specific products.
